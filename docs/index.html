0x00  What it is

zbrowser is a Chromium/Blink browser, rebranded. On every launch it preloads the zpwrchrome power-tool, a cyberpunk new-tab page, and an internal-HUD skin with an 8-scheme picker — running against a dedicated profile so it never touches your system Chrome. Two paths: a runtime rebrand (default, nothing to compile) and an optional source fork that restyles the native chrome and DevTools.

0x01  Why a real Blink base

zpwrchrome is a Manifest V3 extension needing userScripts, declarativeNetRequestWithHostAccess, nativeMessaging and a service-worker background. That runs only on a genuine Chromium engine — not WebKit (Tauri/Safari) or Servo.

The base is a plain Chromium snapshot — unbranded, no Google logo, no "for automated testing" banner (that stripe is exclusive to Chrome for Testing). It is chosen for one reason: the --load-extension switch that preloads zpwrchrome was removed from branded Chrome in v137 but is kept in Chromium.

0x02  Architecture

Base

Plain Chromium snapshot, pinned rev, fetched by fetch-base.sh.

Rebrand

The base bundle's Dock name & icon patched in place to zbrowser.

New tab

A newtab override: neon clock, omnibox, quick-launch tiles.

Internal HUD

Skins chrome:// pages and adds a floating 8-scheme picker, bridged to a native host.

Power-tool

zpwrchrome loaded as a submodule — reused, never copied.

Launcher

bin/zbrowser starts newtab + zpwrchrome + internal-HUD against ~/.zbrowser/profile.

Fork

Optional source build — a 7-patch series restyles native tabs, fonts, toolbar and DevTools.

0x03  Install

git clone --recurse-submodules https://github.com/MenkeTechnologies/zbrowser.git
cd zbrowser
scripts/install.sh          # fetch base + link `zbrowser` on PATH + rebrand (macOS)
zbrowser                    # launch

Re-run install.sh after a base upgrade.

0x05  Runtime rebrand vs. source fork

A solo-maintained Chromium source fork means re-merging security patches against a codebase that changes daily — the trap that keeps Brave and ungoogled-chromium team-sized. So the default path is the runtime rebrand: HUD new-tab + internal-HUD skin + your extensions on a prebuilt Chromium snapshot, upgraded by swapping one binary, nothing to compile.

A theme extension changes colors only — it can't reshape tabs, fonts, or the toolbar (native C++). For the whole chrome in the HUD, the optional fork/ path compiles a patched Chromium: a 7-patch series (sharp tabs, cyberpunk palette, mono UI font, neon toolbar, sharp omnibox, zbrowser strings, and 8 HUD schemes in the DevTools Theme dropdown) authored against the pinned tag and verified apply-clean. It costs a ~100 GB checkout, a 1–4 hr first build, and ongoing rebase maintenance — see fork/.
